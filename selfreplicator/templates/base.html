Requirements:

When creating a Python environment, or deploying to Heroku, these requirements will be installed automatically from requirements.txt

Note: The python libraries will also install their own required libraries.

  • Python 3.7
  • Django 2.2.1
  • gunicorn 19.9.0
  • django-heroku 0.3.1

Installation

  • Install Python 3.7 from the download page here
  • Download the code from GitHub or using git and place it into a new folder
  • Open the settings file, and change the
  • Open a command prompt and navigate to this folder, then type: python -m venv venv to create a python virtual environment, then pip install -r requirements.txt It will automatically install required libraries from requirements.txt
  • In that same command prompt, type python manage.py runserver to run the app using Django's development server, to verify the project is set up correctly
  • Create a new OAuth app on GitHub for this project here which will give you the necessary client IDs to connect to GitHub's API
  • Set the "Authorization callback URL" to the results page of the app (warning: if running locally on the dev server, this callback URL won't work correctly because the localhost is not a valid URL for this field)
  • In the settings file for the project, add the client ID and client secret ID assigned by GitHub to the variables in this file.
  • To run the server, use the python manage.py runserver command from the project directory and with that, you should be ready to try the app out for yourself!
  • To deploy this app using Heroku you can check out their Getting Started guide

How it Works

This application was created with Python using Django and deployed on Heroku

Clicking the button sends the requests to access the user's GitHub profile and creates a repository for the user with this application's own code in it. It does not require the users's password or access to the user's private repositories. The request is made via a url to GitHub using the client ID assigned to this app after registering it with GitHub, and once the user has logged in, asks if the user will accept the app's access request. Agreeing sends the user to the "callback" url provided to GitHub when registering the app.

Sending along with the callback url, GitHub also provides a code that can be exchanged for an authorization token, which allows the app to use the authorization the user agreed to in order to access the public repositories.

Once the code is sent to GitHub's API with a request for an auth token, the token is then used to get the authenticated user's username, which will be used when pushing the files to the repository that will be created.

The repository is created in the user's public repositories, using an request sending the name of the repository, description, homepage, and an option telling it to not auto-initialize with a readme file, since we already have one in the project.

Once the repo has been created, we use the same auth token and the user's username in a request to push content to that new repo.
